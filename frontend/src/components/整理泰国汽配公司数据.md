✅ **修复成功！**

**问题根因分析：**

在 `BaseTable.vue` 分页组件中，事件绑定错误导致翻页功能失效：

| 位置 | 问题描述 |
|------|----------|
| 第84行 | `@current-change` 绑定到了 `handleCurrentChange` |
| 第120行 | `handleCurrentChange` 发出的是 `current-change` 事件 |
| 第39行 | `CompanyList.vue` 监听的是 `@page-change` 事件 |
| 第136行 | 正确的方法是 `handleCurrentPageChange`，它发出 `page-change` 事件 |

**修复内容：**

将 `BaseTable.vue:84` 的事件绑定从：
```vue
@current-change="handleCurrentChange"
```
改为：
```vue
@current-change="handleCurrentPageChange"
```

**修复原理：**
- `el-pagination` 的 `@current-change` 事件在用户点击页码或翻页按钮时触发
- 原来绑定到错误的方法 `handleCurrentChange`，它发出的是表格行选中事件（`current-change`），而非分页事件
- 修复后绑定到正确的 `handleCurrentPageChange` 方法，它会发出 `page-change` 事件
- `CompanyList.vue` 监听到 `page-change` 事件后调用 `loadData()` 重新加载数据

- ✅ 项目构建成功（`npm run build:prod`）
- ✅ 分页事件链路完整：用户点击翻页 → `el-pagination` 触发 `current-change` → `handleCurrentPageChange` 发出 `page-change` → `CompanyList.loadData()` 加载新页数据
